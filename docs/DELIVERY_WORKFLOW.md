# 服务器开发与生产验收流程

> 状态：active
> 最后核验：2026-07-16
> 适用范围：极速蹬 Fork 的功能、修复、配置、迁移和上游同步

本文档是极速蹬长期交付流程的权威说明。服务器负责隔离开发、自动化验证、提交、推送和部署核对；用户本地电脑浏览器负责最终生产验收。旧文档中的“仅服务器线上验收”表述均由本流程替代。

## 生产不变量

- 生产源码分支是 `origin/play/main`。
- `origin/play/main` 触发 Zeabur 生产部署。
- 生产验收地址是 `https://www.jisudeng.com/`。
- 禁止 rebase 或强推 `play/main`。
- 服务器浏览器、本地启动服务和 `localhost` 均不能作为最终产品验收。

## 1. 隔离开发

在服务器上从最新审查基线创建独立分支和 Git worktree。一个交付任务只在自己的 worktree 中修改，避免污染共享工作树或覆盖并行改动。

开始前记录：

- 基线分支和 commit。
- 工作分支和 worktree 路径。
- 需求、验收标准及受影响的 `FORK-*` 条目。
- 数据库、API、权限、计费和部署风险。

## 2. TDD 与双重审查

所有业务行为严格执行：

1. `RED`：先添加能证明需求或回归的失败测试，并确认失败原因正确。
2. `GREEN`：实现满足测试的最小业务修改。
3. `REFACTOR`：在测试持续通过的前提下整理实现。
4. `规格审查`：逐项核对需求、范围和验收标准，修复遗漏后复审。
5. `代码质量审查`：检查正确性、安全性、并发、边界、可维护性和测试质量，修复问题后复审。

每个任务必须依次通过规格审查和代码质量审查，不能用最终总审查替代逐任务审查。纯文档或机械变更可不制造无意义测试，但仍需双重审查和适用的静态验证。

## 3. 服务器验证闸门

服务器允许且必须运行测试和构建。提交和推送前，至少执行：

```bash
./scripts/check-fork-integrity.sh
make test
make build
```

同时运行本次修改对应的定向测试、lint、类型检查或迁移验证。任一命令失败时先修复并重跑，不得提交、推送、部署或宣称完成。

不得为最终验收启动 `go run serve`、`pnpm dev`，不得使用 localhost 或服务器浏览器代替生产验收。自动化测试和 production build 不受此限制。

## 4. 提交、审查分支与合并

验证闸门通过后：

1. 检查 diff，只暂存当前任务文件，确认没有凭据或无关改动。
2. 创建可追溯提交并记录 commit SHA。
3. 先推送 `codex/**` 审查分支，等待规格、代码质量和 GitHub CI（包括 Fork Integrity）确认。
4. 确认后以 merge commit 或其他非 rebase、非强推方式进入 `play/main`。
5. 推送 `origin/play/main`；禁止向 `origin/main` 发布极速蹬生产代码。

不得绕过审查分支直接把未经确认的开发提交推入生产分支。若使用其他审查分支命名，必须创建目标为 `play/main` 的 PR，以触发同等 CI 闸门。

## 5. Zeabur 部署核对

`origin/play/main` 推送后等待 Zeabur 完成生产部署。验收前必须记录并确认：

- 预期的 `play/main` commit SHA。
- Zeabur 实际部署的 commit SHA 与预期一致。
- 构建和发布状态成功。
- 生产健康检查通过，关键 API 无异常。

部署 commit 不一致、状态未完成或健康检查失败时不得开始验收，也不得宣称交付完成。

## 6. 用户本地电脑浏览器验收

部署健康后，由用户在本地电脑浏览器打开 `https://www.jisudeng.com/` 完成最终验收，至少覆盖：

| 身份 | 必查内容 |
|------|----------|
| 游客 | 公共首页、公开路由、登录前可见性和权限边界 |
| 普通用户 | 本次用户功能、导航、状态变化、错误提示和权限边界 |
| 管理员 | 对应管理页面、配置、审计或数据变化，以及普通用户不可见的权限边界 |

功能同时涉及用户页和管理员页时，两侧必须闭环检查。视觉或主题相关修改检查浅色与深色；数据、奖励、余额、计费、统计、配置或迁移相关修改按需进行 UI、API 和数据库数字对账。

服务器端自动化、服务器浏览器截图、curl 或健康检查只能作为部署和技术证据，不能替代用户本地电脑的三身份浏览器验收。若当前无法操作用户本地电脑，应明确记录“等待本地浏览器验收”，不能写“已完成”。

## 7. 凭据安全

- 登录密码、API key、Cookie、数据库连接串和生产凭据只通过环境变量或用户人工输入提供。
- 不在 Git、代码、文档、测试夹具、命令参数、命令输出、日志、截图文件名或验收记录中保存秘密。
- 验收证据只记录身份类型、结果和脱敏标识，不记录密码、令牌或完整 Cookie。
- 怀疑凭据泄露时立即停止传播，撤销并轮换对应凭据，再清理受影响记录。

## 8. 完成证据

每次功能完成必须留下以下记录，敏感字段一律脱敏：

```text
交付范围：
审查分支：
交付 commit：
规格审查：
代码质量审查：
定向测试：
完整测试：
完整构建：
Fork integrity：
origin/play/main commit：
Zeabur 部署 commit：
Zeabur 健康状态：
本地浏览器游客验收：
本地浏览器普通用户验收：
本地浏览器管理员验收：
浅色/深色验收（如适用）：
API/数据库对账（如适用）：
遗留问题：
```

只有所有适用项通过并有证据时才能宣称完成。失败、跳过、等待或无法核实时，必须如实标记并继续处理或请求必要授权。
