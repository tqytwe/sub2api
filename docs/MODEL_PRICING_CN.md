# 模型与价格 — 管理员配置指南

## 开关语义

| 设置项 | 作用 |
|--------|------|
| **公开模型目录** (`public_models_enabled`) | 游客访问 `/models` 时是否展示价目表 |
| **启用登录价目** (`available_channels_enabled`) | 登录用户 `/models` 与 `GET /channels/available`；关闭后登录页显示「暂未开放」 |

极速蹬用户侧栏入口为 **「模型与价格」**（`/models`），不恢复「可用渠道」文案。

## 四层价格关系

1. **AIHubMix 官方参考价** — 同步后写入独立的 `official_*` 字段，只用于对比，不参与扣费。
2. **本站参考价** — 管理员可手工填写四类价格，或设置为“AIHubMix 官方价 × 本站倍率”。
3. **渠道定价**（`/admin/channels/pricing`）— 实际计费基础价与模型映射；同步官方价不会修改渠道价。
4. **分组倍率**（`/admin/groups`）— 登录用户实付价 = 渠道价 × `rate_multiplier`（或用户专属倍率）。

模型价目管理（`/admin/model-catalog`）还负责模型展示名称、排序、游客可见和登录可见。游客与登录用户看到的本站价均为参考价，不能替代实际账单价格。

## 模型协议与登录价目分组

模型的 `platform` 表示调用协议或模型供应商类型，不是业务分组。例如 Codex、国产分组和订阅分组都可能使用 `openai` 协议，仅凭 `platform=openai` 无法判断模型应该展示在哪个分组。

管理员可在模型价目新增、编辑、批量操作和发现池导入时设置“登录价目分组”：

| 模式 | 行为 |
|------|------|
| **仅指定分组** | 登录价目只在选中的真实分组中展示，不会补到其他同协议分组。国产模型应明确绑定到“国产分组”。 |
| **按平台自动匹配** | 保留旧数据兼容行为，按 `platform` 自动匹配；同一协议存在多个业务分组时可能产生歧义，仅建议用于历史数据或分组结构非常简单的实例。 |

数据库字段 `site_model_catalog.group_ids` 的语义为：`NULL` 表示按平台自动匹配，非空数组表示仅允许指定分组。管理后台主表会直接显示当前绑定的分组名称，分组后续改名时自动读取最新名称，不是写死文案。

## 同步「刷新模型数据」

- 管理后台 → 模型价目 → **刷新模型数据**
- 异步 Job，约 1.5s 轮询完成
- AIHubMix 是唯一允许的同步源，响应价格单位为 USD / 1M Token；入库时只转换一次为项目内部的 USD / Token，展示时再还原为每百万 Token 价格
- 已在目录中的模型只更新独立的 AIHubMix 官方价；**新模型进入发现池**，需管理员搜索、勾选后导入
- 导入时必须确认登录价目分组，并可指定本站倍率；导入后可切换为手工本站价，或继续使用“官方价 × 本站倍率”
- 官方价刷新后，倍率模式的本站价自动重算；手工本站价保持不变
- 源失败时不删除模型、不修改官方价、本站价或渠道计费价，并在同步任务中保存真实错误

环境变量（可选）：

| 变量 | 说明 |
|------|------|
| `MODEL_SYNC_SOURCE` | 固定为 `aihubmix`（默认）；其他值会拒绝同步 |
| `AIHUBMIX_MODELS_URL` | 默认 `https://aihubmix.com/api/v1/models` |
| `AIHUBMIX_MODELS_FILE` | 本地 JSON 缓存路径 |
| `AIHUBMIX_API_KEY` | 可选；AIHubMix 接口需要鉴权时发送 Bearer Token |

## 用户验收

1. 游客 `/models`：仅展示「游客可见」模型和官方价/本站参考价。
2. 登录 `/models`：展示全部「登录可见」模型与本站参考价；当前渠道/分组可用时再展示分组和实付价，未匹配渠道时显示 `—`。
3. 管理后台手工修改本站价后，官方价和渠道价不变。
4. 管理后台设置本站倍率并刷新 AIHubMix 后，本站参考价随官方价重算，渠道价不变。
5. 实付价与一次真实 API 调用扣费一致（抽样验证）。
6. 将两个同为 `openai` 协议的业务分组分别绑定不同模型后，登录价目不得跨分组补齐；模型绑定的分组改名后，页面应显示新名称。
