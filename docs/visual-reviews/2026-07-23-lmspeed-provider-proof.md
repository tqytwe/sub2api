## Scope

This review covers the new LMSpeed public provider proof section on the Jisudeng home page. The change embeds only public OG image URLs for `health`, `models`, and `recent`, and leaves API-key speed testing inside the existing authenticated `/keys/speed-test` flow.

## Baseline

The baseline home page only showed the small LMSpeed listing badge in the footer. Public visitors could not see the provider health, supported-models, or recent-test cards without visiting LMSpeed directly.

Baseline artifact: `docs/visual-reviews/assets/lmspeed-provider-proof/baseline-lmspeed-provider-proof.png`.

## Prototype

The prototype places a compact third-party status section after the statistics strip and before the Image API section. The layout uses one larger health card beside two smaller table cards, then collapses to one column on mobile.

Prototype artifact: `docs/visual-reviews/assets/lmspeed-provider-proof/prototype-lmspeed-provider-proof.png`.

Approval status: user approved deploying LMSpeed embedding, with the product boundary that API-key speed testing remains under our control.

Scope boundary: public LMSpeed proof images and safe provider links only. No API key, `apiKey=`, `baseUrl=`, model parsing, chat-completion parsing, or `/keys/speed-test` behavior is changed.

## Reuse Decision

The implementation reuses the existing home section structure, typography classes, `page-container`, and dark-mode CSS variables. It adds a narrow `LmspeedProviderProof` component rather than expanding the footer badge or mixing with API Key modal code.

## State Coverage

Default state renders the section title, bilingual lede, three public image cards, and a provider-page link. Hover state only changes border, shadow, small vertical position, and link gap. Focus-visible keeps a current-color outline. Loading state relies on browser lazy image loading with stable width and height attributes.

## Viewport Coverage

The static board covers the desktop arrangement and a mobile single-column inset. The CSS uses a single-column breakpoint below 900px, while the existing home page mobile section spacing continues below 767px.

## Residual Risk

The artifacts are static review boards, not browser screenshots. After deployment, the public home page should still be checked on the user's local machine in Chinese and English, and clicking the LMSpeed provider cards should be confirmed in a real browser because LMSpeed may challenge non-browser probes.
