实现边界

这块不是独立后台导航，而是套餐配置页里的增长/接入配置面板。

1
管理员维护推荐规则 选择分组、套餐、排序、文案、余额条件和 CTA。
2
前台实时交叉校验 读取可用分组、结算配置、用户余额、订阅状态，过滤掉不能执行的动作。
3
用户创建 Key 前先被教育和转化 低余额去充值，订阅型分组先买套餐，有资格才进入创建 Key。
保存前必须验证分组和套餐是否仍存在；已删除、下架、关闭的配置不能展示给用户。
用户侧效果 API Keys 空状态顶部出现“推荐接入路线”，但不改变原 Key 列表的主结构。
后台侧效果 仍从“订单管理 / 套餐配置”进入，不让左侧导航继续膨胀。
